HTML 中的 href 和 src 虽然都用于指定外部资源的路径，但它们在核心作用、加载行为和应用场景上有本质区别。

| 对比维度 | src (Source) | href (Hypertext Reference) |
| :------ | :------------| :------------------------- |
| 核心作用 | 嵌入资源，成为文档的一部分 | 建立关联，指向外部资源或页面 |
| 加载行为 | 立即加载，可能阻塞文档解析 | 按需加载（如点击链接），或异步加载（如样式表）|
| 适用标签 | `<script>, <img>, <iframe>, <audio>, <video>` 等 | `<a>, <link>, <area>` 等 |


💡 深入理解两者的差异

- src 是“替换”：当浏览器解析到带有 src 属性的标签（如 `<script>`）时，它会暂停当前HTML文档的解析（除非使用 async 或 defer 属性），去下载指定资源并将其内容直接嵌入或应用到当前文档的位置。例如，`<img src="photo.jpg">` 会用图片替换掉这个标签本身 img 和 iframe 资源加载是异步的不会阻塞页面解析。

- href 是“关联”：href 属性则是在当前文档和外部资源之间建立一种连接关系，但不会直接影响当前文档的解析和呈现。例如，`<link href="style.css">` 会让浏览器在后台异步加载CSS文件，同时继续构建HTML结构；而 `<a href="page.html">` 则需用户点击才会触发跳转 。

💎 如何正确选择

一个简单的判断原则是：
- 当你需要将外部资源（如图片、脚本、视频）直接嵌入到当前页面中，成为其一部分时，使用 src。

- 当你需要链接到另一个页面、样式表，或者定义一个导航关系时，使用 href 。
