Canvas 和 SVG 是两种在网页上绘制图形的核心技术，它们在工作原理和适用场景上有着根本的不同

| 特性               | Canvas                                                       | SVG                                                          |
| ------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **类型**           | **位图（像素）**                                             | **矢量图**                                                   |
| **渲染方式**       | 通过 JavaScript API **动态绘制像素**，绘制后不保留图形对象信息 | 使用 **XML 描述图形**，每个图形都是独立的 DOM 节点           |
| **缩放效果**       | **依赖分辨率**，放大后会失真、模糊                           | **分辨率独立**，无限缩放都不会失真                           |
| **交互性**         | **不支持内置事件处理器**，需手动计算坐标实现交互，实现复杂   | **原生支持事件处理器**，可为每个图形元素绑定点击、悬停等事件 |
| **性能特点**       | 适合**高频重绘**（如游戏、动画），性能高，但复杂度随画面复杂度增加 | 适合**静态或少量动态图形**，大量DOM节点会导致性能下降        |
| **文本与可访问性** | 文本渲染能力较弱，可访问性支持较差                           | 文本独立、可编辑、可搜索，可访问性支持好                     |

### 💡 如何选择？

了解了基本区别后，你可以根据具体项目需求来做决定：

- **选择 Canvas 的情况**：当你的项目涉及**大量像素操作、高频动态渲染或复杂的图像处理**时。典型场景包括： **动态游戏**：需要达到60FPS流畅帧率的游戏 。 **数据可视化**：实时更新的股票走势图、包含数万个数据点的散点图或热力图 。 **图像处理与滤镜**：在线图片编辑器、为照片或视频添加实时特效 。
- **选择 SVG 的情况**：当你的项目**需要图形无损缩放、复杂的用户交互，或者对可访问性有要求**时。典型场景包括： **可缩放图形**：企业Logo、图标、需要适应不同屏幕尺寸的插图 。 **交互式图表和地图**：可点击的区域地图、组织架构图，用户需要与图形的各个部分进行交互 。 **需要SEO和可访问性支持的图形**：图形中的文本内容需要被搜索引擎抓取或被屏幕阅读器识别 。

### 🛠️ 进阶技巧与混合使用

在一些复杂的应用中，你甚至可以结合两者，取长补短：

- **混合使用**：例如，在一个数据仪表盘中，可以使用 **Canvas 来高效绘制底层承载大量数据点的趋势图**，同时使用 **SVG 在顶层覆盖交互元素**（如提示框、可点击的图例）。这样既保证了渲染性能，又获得了灵活的交互能力 。
- **性能优化**： **Canvas**：可以通过**分层渲染**（将静态背景和动态内容分别绘制在不同的画布上）来避免不必要的重绘 。对于计算密集型任务（如图像滤镜），可以考虑使用 `OffscreenCanvas`在 Web Worker 中处理，避免阻塞主线程 。 **SVG**：应尽量减少DOM节点数量，例如，用单个 `<path>`元素替代多个 `<line>`元素 。对需要动画的元素使用 `will-change: transform`属性可以触发GPU加速，提升流畅度 。
